# AI Evaluation and Observability Skill

Use this skill when making AI systems measurable and debuggable.

## Observability Requirements
- trace agent steps
- log prompts and tool calls safely
- track latency
- track token usage/cost if relevant
- record failure reasons
- capture final outputs and confidence/quality signals where possible

## Evaluation Types
- unit-style prompt tests
- workflow tests
- retrieval quality tests
- tool-call correctness tests
- human review
- production monitoring

## Metrics
- success rate
- task completion rate
- hallucination rate
- tool error rate
- average retries
- latency
- cost
- user satisfaction

## Guidance
- Add observability from day one
- Evaluate each subsystem separately
- Keep benchmark datasets versioned
- Review failures for pattern detection

## Deliverables
- instrumentation plan
- evaluation criteria
- sample test cases
- dashboard / logging recommendations
