# Backend Architecture Skill

Use this skill when structuring backend systems.

## Principles
- separate transport, business, and persistence layers
- keep domain logic away from framework glue
- design for testability
- keep external integrations isolated

## Concerns to Address
- service boundaries
- transaction/data consistency
- retries and idempotency
- logging and monitoring
- config management
- security
- scaling

## Deliverables
- module boundaries
- dependency flow
- failure handling approach
- observability design
- migration notes if refactoring
