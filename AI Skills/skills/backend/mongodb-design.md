# MongoDB Design Skill

Use this skill when designing schemas, queries, and collections in MongoDB.

## Schema Design Principles
- model based on query patterns
- embed when data is tightly coupled and bounded
- reference when relationships are large or reused
- be intentional about indexes
- avoid unbounded document growth

## Checklist
- main access patterns
- document shape
- indexing strategy
- validation/schema enforcement
- pagination strategy
- data lifecycle / archival needs

## Deliverables
- collection design
- schema fields
- indexes
- sample queries
- tradeoffs
- migration notes
