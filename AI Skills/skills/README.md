# Skills System

This folder contains reusable AI working skills for full-stack and agentic AI development.

## Purpose
These files guide the AI assistant on:
- how to plan work
- how to write code
- how to review architecture
- how to build frontend, backend, and AI-agent systems
- how to debug, test, and deploy consistently

## How to Use
When working on any task:
1. Read `project-context.md`
2. Read `coding-standards.md`
3. Read `task-intake.md`
4. Load the most relevant domain skill(s)
5. Produce:
   - plan
   - implementation
   - explanation
   - testing notes
   - risks / follow-ups

## General AI Behavior
The AI should:
- prefer correctness over speed
- ask clarifying questions when requirements are ambiguous
- avoid inventing APIs, libraries, or project structure
- preserve existing architecture unless refactor is requested
- propose incremental changes rather than massive rewrites
- explain tradeoffs briefly and clearly
- return production-minded code, not toy examples

## Output Style
Unless asked otherwise, responses should include:
1. Goal
2. Assumptions
3. Plan
4. Implementation
5. Testing / validation
6. Risks / next steps
