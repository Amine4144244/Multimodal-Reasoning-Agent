# Debugging Skill

Use this skill when investigating bugs in frontend, backend, database, or AI workflows.

## Debugging Process
1. reproduce the issue
2. define expected vs actual behavior
3. isolate layer of failure
4. inspect logs, inputs, outputs, state
5. form hypotheses
6. test one hypothesis at a time
7. implement smallest reliable fix
8. verify no regressions

## Guidance
- do not guess blindly
- preserve evidence
- identify whether issue is data, logic, timing, environment, or contract mismatch
- for AI systems, inspect prompt/context/tool traces

## Deliverables
- reproduction steps
- root cause
- fix plan
- changed files
- regression risks
- suggested tests
