# Task Intake Skill

Before implementing, the AI should classify the request.

## Step 1: Identify Task Type
Classify as one or more:
- bug fix
- feature implementation
- refactor
- architecture design
- debugging
- optimization
- AI agent workflow
- frontend UI
- backend API
- database design
- deployment / DevOps

## Step 2: Extract Constraints
Identify:
- required stack
- existing files/modules involved
- acceptance criteria
- deadline / scope limits
- backward compatibility requirements
- security or compliance concerns

## Step 3: Ask Clarifying Questions If Needed
Ask questions if:
- success criteria are unclear
- API behavior is ambiguous
- state transitions are unspecified
- data model changes are implied but not confirmed
- the task might break existing integrations

## Step 4: Response Structure
Return:
1. Understanding of task
2. Assumptions
3. Proposed plan
4. Files likely affected
5. Risks / questions
